# Модуль «Личный кабинет» — как проверить

Экраны «Мои занятия» и «Мои абонементы» с записями, листом ожидания и абонементами.

## Что добавлено

Backend (`supabase/migrations/0004_account.sql`):
- `get_my_schedule()` — записи + лист ожидания текущего пользователя в одном списке (с флагом `is_past`).
- `get_my_memberships()` — абонементы пользователя с названием тарифа.

Flutter (Clean Architecture):
- **domain**: `MyScheduleItem`, `Membership`, enum'ы `MembershipKind` / `MembershipStatus`, интерфейс `AccountRepository`.
- **data**: `AccountMappers`, `AccountRepositoryImpl` (RPC).
- **presentation**: `myScheduleProvider`, `myMembershipsProvider`, `AccountController` (отмена записи / выход из очереди + инвалидция), экраны `MyBookingsScreen` (вкладки «Предстоящие» / «Прошедшие») и `MyMembershipsScreen` (активные / история).
- Профиль: пункты «Мои занятия» и «Мои абонементы»; маршруты `/my-bookings`, `/my-memberships`.

## Применить миграцию

В Supabase SQL Editor выполнить `0004_account.sql` (после 0001–0003).

## Сценарий проверки (на seed-данных)

Войти как **client1@yoga.test** (пароль `Passw0rd!`):
- Профиль → «Мои занятия» → вкладка «Предстоящие»: занятие в 11:00 со статусом «Записан» и позиция «В очереди» на занятие в 19:00.
- Отменить запись на 11:00 → элемент исчезает, расписание на экране «Расписание» тоже обновляется (общая инвалидция).
- «Выйти из очереди» на занятии 19:00 → позиция снимается.
- Профиль → «Мои абонементы»: активный «Абонемент 8 занятий», прогресс-бар (осталось 6 из 8), дата окончания.

Войти как **client3@yoga.test**: в абонементах — «Безлимит на месяц» (без прогресс-бара).

## Замечания по архитектуре

- Управление состоянием — **Riverpod 3** (`FutureProvider` + `AsyncNotifier`). `StateNotifier` намеренно не используется (legacy в Riverpod 3).
- Авто-обновление между расписанием и кабинетом — через `ref.invalidate`. При желании позже можно заменить на Supabase Realtime-подписку на таблицу `bookings`.
- Отмена записи переиспользует общий RPC `cancel_booking` (через `ScheduleRepository`), который сам переносит первого из листа ожидания.
